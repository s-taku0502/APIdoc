API 用語集

基本概念

API

Application Programming Interface

アプリケーションやシステム同士が情報をやり取りするための仕様や規約。異なるソフトウェア間での通信を可能にします。

例：WebアプリケーションからTwitter APIを使用して、ツイートを投稿する

エンドポイント

Endpoint

APIにアクセスするための特定のURL。それぞれのエンドポイントは特定の機能やリソースに対応します。

例：https://api.example.com/users - ユーザー情報に関するエンドポイント

REST

Representational State Transfer

WebAPIを設計する際の主要なアーキテクチャスタイル。リソースベースでHTTPメソッドを使用した設計方法です。

例：GET /users（ユーザー一覧取得）、POST /users（新規ユーザー作成）

HTTPメソッド

GET

リソースの取得に使用されるメソッド。データの読み取りのみを行い、サーバーの状態を変更しません。

例：GET /api/articles - 記事一覧の取得

POST

新しいリソースの作成に使用されるメソッド。サーバーにデータを送信します。

例：POST /api/users - 新規ユーザーの作成

PUT

既存のリソースの更新に使用されるメソッド。リソース全体を置き換えます。

例：PUT /api/articles/123 - 記事の全体更新

PATCH

リソースの部分的な更新に使用されるメソッド。

例：PATCH /api/users/456 - ユーザー情報の一部更新

DELETE

リソースの削除に使用されるメソッド。

例：DELETE /api/posts/789 - 投稿の削除

レスポンス関連

ステータスコード

Status Code

APIリクエストの結果を示す3桁の数字。主要なものは以下の通り：

  • 200: OK（成功）
  • 201: Created（作成成功）
  • 400: Bad Request（不正なリクエスト）
  • 401: Unauthorized（認証エラー）
  • 403: Forbidden（アクセス権限なし）
  • 404: Not Found（リソースが見つからない）
  • 500: Internal Server Error（サーバーエラー）

レスポンス形式

Response Format

APIからのデータ返却形式。一般的にJSON形式が使用されます。

{
    "status": "success",
    "data": {
        "id": 123,
        "name": "例文太郎"
    }
}

認証とセキュリティ

APIキー

API Key

APIへのアクセスを制御するための文字列。リクエストヘッダーやクエリパラメータとして送信されます。

例：X-API-Key: abcd1234efgh5678

JWT

JSON Web Token

認証情報を安全にやり取りするための規格。トークンベースの認証に使用されます。

例：Authorization: Bearer eyJhbGciOiJS...

CORS

Cross-Origin Resource Sharing

異なるドメインからのAPIリクエストを制御するセキュリティ機能。

例：Access-Control-Allow-Origin: *

その他の重要概念

レート制限

Rate Limiting

一定時間内のAPIリクエスト数を制限する機能。サーバーの負荷を制御します。

例：1分間に100リクエストまで

ページネーション

Pagination

大量のデータを複数のページに分割して返す機能。

例：/api/posts?page=2&limit=20

キャッシュ

Cache

頻繁にアクセスされるデータを一時的に保存し、レスポンス時間を短縮する機能。

例：Cache-Control: max-age=3600
